## 核心概念

### 1. Agent 为主

- AI Agent 是系统的核心角色
- 负责主动与用户互动
- 智能推荐配对
- 生成个性化消息

### 2. 人为辅

- 人类用户仅负责：
  - 配置系统参数
  - 查看用户数据
  - 处理举报

## Agent 交互流程

### 完整流程图

```
用户注册 ──► Agent 问候 ──► 获取推荐 ──► 用户反馈 ──► 配对
   │           │             │            │            │
   ▼           ▼             ▼            ▼            ▼
┌──────┐   ┌────────┐   ┌────────┐   ┌────────┐   ┌──────┐
│入口   │   │首次互动 │   │浏览推荐 │   │喜欢/   │   │配对  │
│      │   │        │   │        │   │不喜欢   │   │成功  │
└──────┘   └────────┘   └────────┘   └────────┘   └──────┘
                                                  │
                                    ┌────────────┴────────────┐
                                    ▼                         ▼
                             ┌──────────┐              ┌──────────┐
                             │ 跟进用户  │              │ 成功案例  │
                             └──────────┘              └──────────┘
```

### 详细步骤

#### 第一步：用户注册/首次接触

```
Agent: "你好呀！我是你的AI红娘，很高兴认识你！"
       "在开始之前，先让我了解一下你吧~"
```

**API 调用：**
```bash
POST /api/agent/action
{
  "action": "message",
  "userId": "user123",
  "params": {
    "type": "greeting"
  }
}
```

#### 第二步：获取今日推荐

```
Agent: "为你精选了几位优质对象，来看看有没有心仪的吧~"
       [展示推荐卡片列表]
```

**API 调用：**
```bash
POST /api/agent/action
{
  "action": "recommend",
  "userId": "user123"
}
```

**响应包含：**
- 推荐用户列表
- 匹配分数
- 匹配原因
- Agent 消息

#### 第三步：用户反馈

用户对推荐进行操作：
- 👍 喜欢
- 👎 不喜欢
- ❤️ 超喜欢

**API 调用：**
```bash
POST /api/recommendations/{recommendationId}/feedback
{
  "userId": "user123",
  "action": "like"
}
```

#### 第四步：配对

当双方都"喜欢"对方时，触发配对：

```
Agent: "太好了！发现你们彼此喜欢~"
       "缘分到了！准备开始聊天吧~"
```

**API 调用：**
```bash
POST /api/agent/action
{
  "action": "match",
  "userId": "user123",
  "params": {
    "targetUserId": "user456"
  }
}
```

#### 第五步：跟进

Agent 定期跟进用户：

```
Agent: "最近怎么样？有没有遇到合适的TA呀？"
```

**API 调用：**
```bash
POST /api/agent/action
{
  "action": "followup",
  "userId": "user123"
}
```
